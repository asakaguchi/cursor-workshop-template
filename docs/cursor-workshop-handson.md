# Cursor×Python×GitHub 学習ガイド

## AI とプログラミングを始めよう

### 学習のゴール

**プログラミングが全く初めてでも大丈夫です！** この学習コンテンツでは、以下のことを体験していきます：

1. **AI と会話しながらコードを書く** - 「これがしたい」と伝えるだけでOK

2. **アプリを作る楽しさを知る** - 商品を登録して見る簡単なWebアプリを作ります

3. **プロのような開発方法を学ぶ** - GitHubでプロジェクト管理をします

4. **インターネットで使える本格アプリ** - 最後はクラウドで公開します

**AI が先生役**になってくれるので、専門用語がわからなくても大丈夫。困ったらその場で聞けます！

## Part 1：環境構築（20分）

## まずは道具の準備をしましょう

プログラミングを始める前に、必要なソフトをインストールします。
**全てコピー＆ペースト**で進められるので、プログラミング初心者でも安心してください。

### 使うパソコン

macOSとWindowsの両方に対応しています。OS固有の手順が必要な場合は、その都度両方の手順を記載しています。

### 何をインストールするの？

- **Cursor**：AI と会話しながらコードが書けるエディタ

- **Python**：プログラミング言語（日本語のような役割）

- **uv**：Pythonの道具を管理するツール（本棚のような役割）

### 1.1 Cursor のインストール

#### インストール手順（5分）

#### Step 1: ダウンロード

1. ブラウザで [cursor.com](https://cursor.com) を開く

2. あなたのOSに対応したダウンロードボタンをクリック
   - **macOS**：「Download for macOS」
   - **Windows**：「Download for Windows」

3. ダウンロードが始まります（しばらく待ってください）

#### Step 2: インストール

**macOSの場合：**

1. ダウンロードフォルダにある「Cursor-x.x.x.dmg」をダブルクリック

2. 開いたウィンドウで、Cursorのアイコンを「Applications」フォルダにドラッグ

3. これでインストール完了です！

**Windowsの場合：**

1. ダウンロードした「Cursor Setup x.x.x.exe」をダブルクリック

2. インストーラーの指示に従って進める

3. これでインストール完了です！

#### Step 3: 起動

**macOSの場合：**

1. Launchpadを開く（Dockの一番左のアイコン）

2. 「Cursor」を見つけてクリック

**Windowsの場合：**

1. スタートメニューから「Cursor」を検索してクリック

2. またはデスクトップのCursorアイコンをダブルクリック

#### 初期設定（10分）

初回起動時に設定を行います。**ゆっくり一つずつ**進めていきましょう。

#### Step 1: ログイン

- 「Sign in with GitHub」をクリック

- GitHubアカウントでログイン（ない場合はその場で作成できます）

#### Step 2: 基本設定

1. テーマ選択：お好みで選んでください（後で変更可能）

2. データ共有の確認：
   - チェックボックスをONにして「Continue」をクリック
   - ⚠️ **重要**：後でプライバシー設定を変更します

#### Step 3: 言語設定

- 「Language for AI」を「**Japanese**」に変更

- 「Install cursor command」をクリック

- 「Continue」をクリック

#### Step 4: プライバシー設定（重要）

1. 右上の⚙️（歯車）アイコンをクリック

2. 「General」→「Privacy」

3. 「**Privacy Mode with Storage**」を選択
   - これで、あなたのコードが学習に使われません

#### Step 5: 日本語化

**macOSの場合：**

1. `Cmd + Shift + X` を押す（拡張機能を開く）

2. 検索欄に「Japanese」と入力

4. 左下に表示される「Change Language and Restart」をクリック

**Windowsの場合：**

1. `Ctrl + Shift + X` を押す（拡張機能を開く）

2. 検索欄に「Japanese」と入力

3. 「Japanese Language Pack for Visual Studio Code」をインストール

4. 左下に表示される「Change Language and Restart」をクリック

### 1.2 Pythonとuvのインストール

**Windowsの場合：**

2. **重要**：「Add Python 3.12 to PATH」に**必ずチェック**を入れる

3. 「Install Now」をクリック

#### uv のインストール（3分）

### uvって何？

Pythonの「道具箱」を管理してくれるツールです。料理で例えると「調味料ラック」のような役割です。

### インストール方法

**Macの場合：**

```bash
# 方法1：公式インストーラー（推奨）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 方法2：Homebrewを使う場合
brew install uv

```

**Windowsの場合：**

```powershell
# PowerShellで実行（推奨）
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

```

#### ⚠️ コマンドの実行方法

**macOSの場合：**

1. Cursorを開く

2. `Cmd + Shift + ~` でターミナルを開く

3. 上のコマンドをコピー&ペーストしてEnterを押す

**Windowsの場合：**

1. Cursorを開く

2. `Ctrl + Shift + ~` でターミナルを開く

3. 上のPowerShellコマンドをコピー&ペーストしてEnterを押す

#### 動作確認

**macOSの場合：**

```bash

# uvがインストールされたか確認

```

**Windowsの場合：**


### 質問が出てきたら、以下のように答える

### 🎉 成功したら

「Authentication complete!」のようなメッセージが表示されます。

### 4.2 TDD の流れを体験

### TDD（テスト駆動開発）って何？

料理で例えると：

1. **📋 レシピを書く**（テスト）

2. **👨‍🍳 料理を作る**（実装）

3. **✨ 味を調える**（改善）

### プログラミングでの3ステップ

1. **🔴 Red**：「こうなってほしい」をテストで書く

2. **🟢 Green**：テストが通る最小限のコードを書く

3. **🔄 Refactor**：コードをより良く改善

### 👀 観察ポイント

AIがこの3ステップを繰り返している様子を見てください。
最初は赤い文字（失敗）が、だんだん緑（成功）に変わります！

### 4.3 テストを動かしてみよう

## 品質チェックを実行

ターミナルで以下のコマンドを実行してみてください：

```bash
# 全体的な品質チェック
uv run pytest

```

### 🔍 詳しく見たい場合

```bash
# 詳細な結果を表示
uv run pytest -v

```

### 🎯 特定の部分だけチェック

```bash
# 特定のテストファイルのみ
uv run pytest tests/test_specific.py -v

```

### 🚦 結果の見方

- **🔴 赤い文字**：まだ作成中（正常です！）

- **🟢 緑の文字**：完成して正常動作

- **❌ FAILED**：何か問題があります

- **✅ PASSED**：完璧に動作しています

### 💡 最初は赤くて当然

TDDでは「まずテストを書いて失敗させる」ので、最初の赤は成功の証拠です！

### 4.4 作業完了の報告

## 完成の確認ポイント

## 困ったときの解決法

### エラーは友達です

プログラミングでエラーが出るのは**当たり前**のことです。
プロの開発者でも毎日エラーと格闘しています。

### エラーメッセージの読み方

エラーは「何が問題なのか」を**親切に教えてくれる先生**です。
英語で書かれていても、AIに翻訳してもらえばすぐに理解できます。

### 解決の基本ステップ

1. **落ち着く**：焦らずにエラーメッセージを読む

2. **コピー**：エラーメッセージを全文コピー

3. **AIに相談**：「このエラーの意味と解決方法を教えて」

4. **試す**：AIの提案を一つずつ試す

5. **コミュニティに相談**：解決しない場合は遠慮なくStack OverflowやGitHub Discussionsで質問

### よくある問題と解決法

### **🚨 よくあるエラーと解決法**

#### **Q：コメント行でエラーが出る**

```text
zsh: command not found: #

```

**🤔 原因**
`#` で始まる行（説明文）を実行してしまっている

##### 解決法

`#` で始まる行は実行しないでください。
**実際のコマンドだけ**をコピー&ペーストしてください。

#### **Q：「brew install sh」でエラー（macOSのみ）**

```text
Warning: No available formula with the name "sh"

```

**🤔 原因**
`sh` ではなく `gh` をインストールする必要がある

##### 正しいコマンド

正しくは `brew install gh` です。
`gh` (GitHub CLI) をインストールしてください。

#### **Q：警告メッセージが表示される**

```text
WARN: The following commands are shadowed by other commands

```

**🤔 これは何？**
「同じ名前のツールが既にあります」という **無害な警告**

**✅ 対処法**
**問題ありません！** そのまま続行してOKです。

#### **Q：GitHub CLI の更新通知**

```text
A new release of gh is available: 2.74.0 → 2.74.1

```

**🤔 更新すべき？**
**学習中は無視してOK**です。

##### ✅ 後で更新したい場合

**macOSの場合：**

```bash
brew upgrade gh

```

**Windowsの場合：**

```powershell
# wingetがある場合
winget upgrade GitHub.cli

# または公式サイトから最新版をダウンロード

```

### **🐍 Python環境の問題解決**

#### **Q：Pythonが見つからない**

```text
command not found: python

```

**🤔 原因**
Pythonがインストールされていない、またはパスが通っていない

##### Pythonの確認方法

**macOSの場合：**

```bash
# まずはpython3を試す
python3 --version

```

**Windowsの場合：**

```powershell
# Pythonを試す
python --version

```

それでもダメなら、Python公式サイトからインストールし直してください。

#### **Q：uvが見つからない**

**🤔 原因**
uvがインストールされていない、またはパスが通っていない

##### uvの再インストール

**macOSの場合：**

```bash
# uvを再インストール
curl -LsSf https://astral.sh/uv/install.sh | sh

# パスを通す
source $HOME/.cargo/env

```

**Windowsの場合：**

```powershell
# uvを再インストール
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# 新しいPowerShellウィンドウを開いて確認

```

#### **Q：仮想環境が有効化されない**

**🤔 原因**
仮想環境の起動コマンドが間違っている

##### 正しい有効化方法

**macOSの場合：**

```bash
source .venv/bin/activate

# 成功したか確認（行の最初に(.venv)と表示される）
which python
# → /プロジェクト/パス/.venv/bin/python と表示されればOK

```

**Windowsの場合：**

```powershell
.venv\Scripts\Activate.ps1

# 成功したか確認（行の最初に(.venv)と表示される）
where python
# → \プロジェクト\パス\.venv\Scripts\python.exe と表示されればOK

```

#### **Q：ポートが使用中のエラー**

```text
OSError: [Errno 48] Address already in use

```

**🤔 原因**
別のアプリが同じポート（8000番）を使っている

##### ポート競合の解決

```bash
# 使っているプログラムを確認
lsof -i :8000

# 終了させる（PIDの番号を確認して）
kill -9 <PID番号>

# または別のポートを使う
uvicorn src.product_api.main:app --reload --port 8001

```

### **🖥️ 画面開発での問題解決**

#### **Q：入力画面でプレビューが更新されない**

**😵 症状**
商品名や金額を入力しても、プレビューが「未入力」のまま

**🤔 原因**
Streamlitの`st.form`の仕様で、ボタンを押すまで入力値が反映されない

##### これは正常な動作

これは**正常な動作**です。登録ボタンを押すと正しく動作します。
リアルタイムプレビューが必要な場合は、フォームの外で入力欄を作る必要があります。

```python
# tests/test_ui_validation.py
def test_商品名の入力チェック():
```
